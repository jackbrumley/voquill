# Voquill v1.3.1

- Completed a broad UI refactor to token-driven, co-located component styling for cleaner long-term maintainability.
- Restored and polished the original visual language across Status, History, Config, and Initial Setup while keeping the new structure.
- Upgraded navigation to route-based views (`#/status`, `#/history`, `#/config`, `#/setup`) to make UI development and deep-link testing easier.
- Unified setting layouts with reusable shared components so Config and Setup use consistent cards, spacing, and control alignment.
- Improved onboarding and diagnostics UX, including clearer logs actions, updated model guide presentation, and stronger setup flow consistency.
- Added deeper overlay status diagnostics and sequencing logs to improve Linux overlay troubleshooting on both X11 and Wayland.
- Redesigned the overlay UI into a unified in-pill status layout and expanded overlay width for cleaner centering and future extensibility.
- Improved overlay rendering stability by reducing problematic transparency/shadow layering while preserving animated orb feedback.
- Added lightweight "✓ Saved" confirmation feedback for debounced config changes to make setting persistence clearer without noisy toasts.
- UI/UX polish: tightened titlebar control hover behavior and spacing, and refined History item typography/layout consistency.
- Startup flow fix: Initial Setup now appears only when required setup checks are incomplete; ready systems launch directly to Status.
